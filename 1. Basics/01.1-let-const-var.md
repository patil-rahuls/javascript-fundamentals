## `let`, `const` & `var`

> **_Use `let` and `var` to mutate primitive data._**

> **_Use `const` for objects and primitives that don't need to mutate/change._**

By default always use `const` if possible so that there are less mutations in the code.
It results in better memory management.

Avoid using `var` which is the old way(ES5) of defining variables as its more error prone. That's the reason the `let` keyword was born!

> **_A `const` variable, while being declared, needs to be initialized._**

```javascript
// âťŚ Incorrect:
const a;
// 'Uncaught SyntaxError: Missing initializer in const declaration'

// âś… Correct:
const b = 2021;
const c = {}; // Allowed for Objects.
const d = []; // Arrays are a special type of Objects.
```

## **`var` variables can be more prone to errors.**

All the following examples work without giving any error, but they look bad and erroneous.

> Example 1: **_`var` variables are initialized with the value `undefined` and are hoisted._**
>
> ```javascript
> var name = "Rahul";
> var name = "Raj";
> // This works without giving any error.
> ```
>
> _Functionally, the code shown above executes like this:_
>
> ```javascript
> var name = undefined;
> name = "Rahul";
> name = "Raj";
> ```

&nbsp;

> Example 2
>
> ```javascript
> b = 25;
> var b;
> console.log(b);
> // 25
> ```
>
> _Functionally, the code shown above executes like this:_
>
> ```javascript
> var b = undefined;
> b = 25;
> console.log(b);
> ```

&nbsp;

> Example 3
>
> ```javascript
> var a = 1;
> var a = 2;
> console.log(a);
> // 2
>
> var a;
> console.log(a);
> // 2
> ```
>
> _Functionally, the code shown above executes like this:_
>
> ```javascript
> var a = undefined;
> a = 1;
> a = 2;
> console.log(a);
> var a; // This does not change the
> // value of 'a' because it is
> // already initialized.
> console.log(a);
> ```

&nbsp;

> Example 4: **_Function declarations are hoisted before any `var` initializer gets evaluated._**
>
> ```javascript
> var a = 1;
> function a() {}
>
> console.log(a);
> // 1
> ```
>
> _Functionally, the code shown above executes like this:_
>
> ```javascript
> // Function declarations are
> // first hoisted.
> function a() {}
>
> // Then var initializers
> // are processed
> var a = 1;
>
> // Value of 'a' is now '1'
> console.log(a);
> // '1'
> ```

&nbsp;

> Example 5
>
> ```javascript
> try {
>   f = 100;
>   throw 1;
> } catch (e) {
>   var f = 2;
> }
>
> console.log(f);
> // 100
> ```
>
> _In the above example, the `var` variable `f` is hoisted > to outside the `catch` block with value `undefined`._
>
> _But any value assigned within the `catch` block is not visible outside._
>
> _Hence it returns the last assigned value which is 100._
