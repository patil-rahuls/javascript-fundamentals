## Function Declaration & Expression - Differences

> **Function declarations** are hoisted before any `var` declaration.
>
> **Function expressions** are not hoisted _(unless they are `var`)_.

Function declarations are hoisted. A function call can be written before its declaration inside a JS file.
```javascript
const area = calcArea(34);
//
// ...
// some logic
// ...

// Function declared later in
// the file.
function calcArea(side) {
  return side**2;
}
```

&nbsp;

Function-expressions are not hoisted.
```javascript
const square = calcSQ(34);
// Error "Cannot Access 'calcSQ' before initialization."

// Function declared later in
// the file.
const calcSQ = function(side) {
  return side**2;
}
```

By contrast, you can only call a function expression in a linear sequence. _You have to define it before you call it_.

> Always prefer function expression because it will enforce us to declare the function before using it, without the `use strict`.
