## `let`, `const` & `var` - Creating Global object property

> **_At the top level, `var` variables create a property on the global object._**
>
> **_`let` and `const` variables do not._**

_Variables declared using `let` and `const` are not accessible to global objects like the `window` or `global` object._

```javascript
var foo = "Foo"; // globally scoped
let bar = "Bar"; // not globally scoped
baz = "Baz"; // globally scoped

// In browser:
console.log(window.foo);
// 'Foo'
console.log(window.baz);
// 'Baz'
console.log(window.bar);
// undefined

// In Node JS runtime:
console.log(global.foo);
// 'Foo'
console.log(global.baz);
// 'Baz'
console.log(global.bar);
// undefined
```

The purpose of `let` and `const` declarations is only to free up memory when not needed in a certain block.
