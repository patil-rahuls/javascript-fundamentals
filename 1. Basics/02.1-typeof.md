## `typeof` Operator

```javascript
console.log(typeof x);
// "undefined" (if x isn't defined)

console.log(typeof "arr");
// "string"

console.log(typeof 4);
// "number"

console.log(typeof true);
// "boolean"

// Despite not being numbers, these
// are considered "number" type.
console.log(typeof NaN);
// "number"
console.log(typeof Infinity);
// "number"

const square = (num) => num * num;
console.log(typeof square);
// "function"

console.log(typeof [2, 4, 6]);
// "object"

console.log(typeof {});
// "object"
```

### How to check if a given object is an Array or an Object?

> **_`Object.prototype.toString.call()`_**

> Example 1 - Array
>
> ```javascript
> const givenVariable = [1, 2, 3];
>
> Object.prototype.toString.call(givenVariable);
> // '[object Array]'
> ```

&nbsp;

> Example 2 - Object
>
> ```javascript
> const givenVariable2 = {
>   name: "Rahul",
>   age: "25",
> };
>
> Object.prototype.toString.call(givenVariable2);
> // '[object Object]'
> ```

### Type Validations _(Optional Read)_

> Example : Check if a given string is number
>
> ```javascript
> function isNumeric(str) {
>   if (typeof str != "string") {
>     // only process strings!
>     return false;
>   }
>
>   return !Number.isNaN(str) && !Number.isNaN(Number.parseFloat(str));
>   // use type coercion to parse the
>   // _entirety_ of the string.
>   // (`parseFloat` alone does not do this)
>   // and ensure strings of whitespace fail.
> }
> ```

_`Number.isNaN` will allow whitespaces whereas `Number.parseFloat`/`Number.parseInt` will not._

```javascript
Number.isNaN(""); // false
Number.isNaN("   "); // false

Number.parseInt(""); // NaN
Number.parseInt("   "); // NaN

Number.parseFloat(""); // NaN
Number.parseFloat("   "); // NaN
```

&nbsp;

_Note that you can also use these number functions directly, without specifying the `Number` class e.g._

```
return !isNaN(str) && !isNaN(parseFloat(str));
```
_This is because these functions have been around since the earliest versions of JavaScript. In ES6 as a part of modularisation, they were directly copied into the `Number` class._

_By specifying the `Number` class while using these functions(e.g. `Number.parseFloat`), it explicitly states your intention to work with numbers and follows a better coding standards._

_Going forward, in these notes, I have specified the `Number` class wherever these methods are used._
