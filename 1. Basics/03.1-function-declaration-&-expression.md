## Function Declaration & Expression

> **_Function declarations_** _are hoisted before any `var` declaration._
>
> **_Function expressions_** _are not hoisted (unless they are `var`)_.

Functions are still Objects in JavaScript _(Callable Objects)_.

## 1. Function declaration

Always hoisted.

Hoisted before any `var` variable declaration.

```javascript
function calcAge(birthYear) {
  return 2021 - birthYear;
}
```

---

&nbsp;

## 2. Function expression

_An expression is something that returns a value._

We can write a function in the form of an expression and assign them to variables.

This is because in javascript, functions are still values just like any other variables.

```javascript
const calcAge2 = function (birthYear) {
  return 2021 - birthYear;
};
```

---

&nbsp;

> _Tricky Question: What will be the output of the folowing code?_
> 
> ```javascript
> var show = function () {
>   return "Hello";
> };
> 
> function show() {
>   return "World";
> }
> 
> show();
> ```
> 
> Output:
>
>      "Hello"
>
> _This is because, the function declaration is first hoisted, and then the function expression is returned to the `var` variable._
