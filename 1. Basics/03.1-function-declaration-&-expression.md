## Function Declaration & Expression

> **Function declarations** are hoisted before any `var` declaration.
>
> **Function expressions** are not hoisted _(unless they are `var`)_.

Functions are still Objects in JavaScript _(Callable Objects)_.

&nbsp;

### 1. Function declaration

> Always hoisted.

> Hoisted before any `var` variable declaration.
```javascript
function calcAge(birthYear) {
  return 2021 - birthYear;
}
```
---

&nbsp;
### 2. Function expression
An expression is something that returns a value.

> We can write a function in the form of an expression and assign them to variables.

This is because in javascript, functions are still values just like any other variables.
```javascript
const calcAge2 = function(birthYear) {
  return 2021 - birthYear;
};
```
---

&nbsp;
> Tricky Question: What will be the output of the folowing code?
```javascript
var show = function() {
  return 'Hello';
};

function show() {
  return 'World';
}

show();
```
> Output:
>
>      "Hello"
> This is because, the function declaration is first hoisted, and then the function expression is returned to the `var` variable.
>
