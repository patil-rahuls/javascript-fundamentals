## Destructuring Assignment [ES6]

Unpacks and assigns values from arrays, or properties from objects into distinct variables.

### Destructuring an Array

```javascript
const tech = ['JavaScript', 'TypeScript', 'Node.js', 'Express', 'React', 'AWS Cloud'];

let first = tech[0];
let second = tech[1];
let third = tech[2];

console.log(first, second, third);
// 'JavaScript' 'TypeScript' 'Node.js'


// Using Destructuring
[first, second, third] = tech;

console.log(first, second, third);
// 'JavaScript' 'TypeScript' 'Node.js'

[, , , fourth] = tech;

console.log(fourth);
// 'Express'
```

> Default Values

When we don't know what and how many elements are there in the array to be destructured, we may end up getting 'undefined'.

```javascript
let [p, q, r] = [8,9];

console.log(p, q, r);
// 8  9  undefined


// Solution : We define Default values while destructuring an array to avoid the 'undefined'.
[p = 1, q = 1, r = 1]  = [8,9];

console.log(p, q, r);
// 8  9  1
```

### Destructuring an Object
>
> Useful in destructuring the APIs data.
>
> _**IMP: We need to specify the correct property name in the destructuring assignment.**_

```javascript
const user = {
  name: "Rahul",
  age: 25,
  city: "Pune"
};

// Specify the same property names of the object
const { name, age } = user;

console.log(name); // "Rahul"

console.log(age);  // 25
```

Incorrect property names will result in undefined

```javascript
let { role } = user;

console.log(role); // undefined
```

Solution: Default/fallback values

In API calls, we might not know whether data will be present or not in the response object. We can assign a Default/Fallback value in case property is undefined or is not present in the object being destructred.

```javascript
const {
  role = "Guest"
} = user;

console.log(role); // "Guest"
```

Renaming variables

```javascript
const { city: location } = user;

console.log(location); // "Pune"
```

### Some other use-cases of destructuring

> Swap two values without using a temporary variable.

```javascript
[first, second] = [second, first];
```

&nbsp;
> Mutate variable values by Destructuring an array.

```javascript
const myArr = [7, 8];
let a = 111, b = 999;

[a, b ] = myArr;

console.log(a,b);
// 7  8
```

&nbsp;

> Mutate variables values by Destructuring an Object
>
> **_Parens Syntax_**

```javascript
const myObj = {
  a: 91 ,
  b: 2 ,
  c: 3 ,
  d: 4 ,
  e: 5
};

let a = 111;
let b = 999;

{ a , b } = myObj;
// Syntax Error
// { a, b } = myObj is not a valid syntax,
// Because, the expresion "{ a, b }" on the left-hand side is considered a 'block' and not an object literal according to the ES6 Specifications.

// Solution: Just enclose the expression in parentheses.
({ a , b } = myObj);

console.log(a,b);
// 91   2
// This is called "parens syntax".
