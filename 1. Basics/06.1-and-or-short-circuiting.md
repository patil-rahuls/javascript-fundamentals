## Short Circuiting `&&` and `||`

> ### The operators || and && do not always return boolean.

## 1. Short Circuiting Logical OR - ||

Keeps evaluating until a 'Truthy' value is found and returns it else returns last value otherwise.

> Example 1

```javascript
console.log(undefined || 0 || "" || 'Hello' || 23 || null);
// 'Hello'
```

In `||` operation, the first truthy value is returned.

It will be _short-circuited_ at the first truthy value `'Hello'`.

&nbsp;
> Example 2

```javascript
console.log(3 || '');
// 3
// 3 is a truthy value, so the next expression is not evaluated

console.log(3 || 'Rahul');
// 3
console.log(true || 0);
// true
```
&nbsp;
> Example 3

```javascript
console.log(undefined || null);
// null
// as no truthy value found, so the last value is returned.
```

## 2. Short Circuiting Logical AND - `&&`

Keeps evaluating until a 'FALSY' value is found and returns it else returns last value otherwise.

> Example 4

```javascript
console.log(undefined && 0 && "" && 'Hello' && 23 && null);
// undefined
```

In `&&` operation, the first falsy value is printed.

It will be _short-circuited_ at the first falsy value `undefined`.

&nbsp;

> Example 5

```javascript
console.log(3 && 'Rahul');
// 'Rahul'
// As no falsy value found, so the last value is returned.
```

&nbsp;
> Example 6

```javascript
console.log(0 && 'Rahul');
// 0
```

&nbsp;
> Example 7

```javascript
console.log('Hello' && 23 && null && "rest");
// null
// The first Falsy value where the short circuit happened.
```

From the above examples, we observed that the `||` and `&&` operations do not always return a boolean, instead they return the data based on the operands (where the short circuiting happens).
