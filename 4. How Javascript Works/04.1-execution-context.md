## Execution Context

An Execution Context is an environment where the code executes.

Invoking a function _(function call)_ prompts the engine to generate a new Function Execution Context and push it to the top of the Call Stack.

Its an environment contains information about the code necessary to execute the code.

> Example: _Demonstration of the execution of a javascript program:_
>
> ```javascript
> let name = "Rahul";
>
> const first = function () {
>   let a = 1;
>   const b = second();
>   a += b;
>   return a;
> };
>
> function second() {
>   var c = 2;
>   return c;
> }
>
> var x = first();
> ```
>
> _After compilation, when the code _(machine code)_ is ready to be executed, following events happen:_
>
> 1.  _Exactly ONE `Global Execution Context (GEC)` is created for the ***top level code*** i.e. all the outermost declarations
>     and definitions except function calls in a source code._
>
>           `Global Execution Context` is pushed onto the bottom of the Call Stack.
>
>     ```
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |        GLOBAL EC       |    |  Call-Stack   |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |                        |    |               |
>     | name <uninitialized>   |    |               |
>     | first <uninitialized>  |    |               |
>     | second = <function>    |    |               |
>     | x = undefined          |    |               |
>     |                        |    |      GEC      |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>
>     ```
>
> 2.  _All the Global scope items are allocated memory. From the above example:_
>
>     _`name`: Allocated memory (`uninitialized` in the ***TDZ***)._
>
>     _`first`: Allocated memory (`uninitialized` in the ***TDZ***)._
>
>     _`second`: The complete function definition is stored directly in memory._
>
>     _`x`: Allocated memory and immediately initialized with the value `undefined`._
>
>     **_only memory is allocated before execution._**
>
> 3.  _Execution of Global code starts:_
>
>     _`const name = "Rahul";` --> Value `"Rahul"` is assigned to variable _`name`_._
>
>     _`const first = function() {...};` --> The function is assigned to the variable _`first`_._
>
>     _`var x = first();` --> Since this is a function invocation, the engine pauses the execution of GEC, creates and pushes the Execution Context of `first()` onto the Call Stack and it then starts executing the function `first()`._
>
>     ```
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |        GLOBAL EC       |    |  Call-Stack   |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |                        |    |               |
>     | name = "Rahul"         |    |               |
>     | first = <function>     |    |               |
>     | second = <function>    |    |               |
>     | x = undefined          |    |               |
>     |                        |    |      GEC      |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>
>     ```
>
> 4.  _Execution of Function Code starts. `first()` is pushed to the stack and run._
>
>     _Just like the GEC, the values and declaration are processed._
>
>     ```
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |        first()         |    |  Call-Stack   |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |                        |    |               |
>     | a = 1                  |    |               |
>     | b = second()           |    |               |
>     | ... paused until       |    |               |
>     | second() finishes      |    |   [first()]   |
>     |                        |    |      GEC      |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>
>     ```
>
>     _`second()` is pushed on the call stack for execution and it sets `c = 2`_
>
>     ```
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |        second()        |    |  Call-Stack   |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |                        |    |               |
>     | c = 2                  |    |               |
>     |                        |    |               |
>     |                        |    |  [second()]   |
>     |                        |    |   [first()]   |
>     |                        |    |      GEC      |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>
>     ```
>
>     _`second()` returns `c` and is popped off the call stack and `first()` resumes_
>
>     ```
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |        first()         |    |  Call-Stack   |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |                        |    |               |
>     | a = 1                  |    |               |
>     | b = second()           |    |               |
>     | a += b                 |    |               |
>     |                        |    |   [first()]   |
>     |                        |    |      GEC      |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>
>     ```
>
>     _`first()` resumes, assigns `b` the returned value of `second()` i.e.`2`, updates a to `3` _(a += b)_, returns `3`, and pops off._
>
>     _Control returns to the Global Execution Context to finish the last line:_
>
>     ```
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |        GLOBAL EC       |    |  Call-Stack   |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |                        |    |               |
>     | name = "Rahul"         |    |               |
>     | first = <function>     |    |               |
>     | second = <function>    |    |               |
>     | x = first()            |    |               |
>     |                        |    |      GEC      |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>
>     ```
>
>     _`var x = first();` --> The engine takes the returned value `3` and overwrites the initial `undefined` value stored in `x`._
>
>     ```
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |        GLOBAL EC       |    |  Call-Stack   |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>     |                        |    |               |
>     | name = "Rahul"         |    |               |
>     | first = <function>     |    |               |
>     | second = <function>    |    |               |
>     | x = 3                  |    |               |
>     |                        |    |               |
>     ||||||||||||||||||||||||||    +++++++++++++++++
>
>     ```

For each function call, a new Function Execution Context is created and stacked on top of the Global Execution Context.

All the function execution contexts in sequence together form a call stack.
