## Execution Context

An Execution Context is an environment where the code executes.

Invoking a function _(function call)_ prompts the engine to generate a new Function Execution Context and push it to the top of the Call Stack.

This environment contains information about the code necessary to execute the code.

&nbsp;

> Example to demonstrate the execution of a javascript program:
```javascript
let name = "Rahul";

const first = function (){
  let a  = 1;
  const b = second();
  a += b;
  return a;
};

function second(){
  var c = 2;
  return c;
}

var x = first();
```

After compilation, when the code _(machine code)_ is ready to be executed, following events happen:

1. Exactly ONE `Global Execution Context (GEC)` is created for the ***top level code*** i.e. all the outermost declarations
and definitions except function calls in a source code.

   `Global Execution Context` is pushed onto the bottom of the Call Stack.
   ```
    ||||||||||||||||||||||||||    +++++++++++++++++
    |        GLOBAL EC       |    |  Call-Stack   |
    ||||||||||||||||||||||||||    +++++++++++++++++
    |                        |    |               |
    | name <uninitialized>   |    |               |
    | first <uninitialized>  |    |               |
    | second = <function>    |    |               |
    | x = undefined          |    |               |
    |                        |    |      GEC      |
    ||||||||||||||||||||||||||    +++++++++++++++++
    ```

&nbsp;

2. All the Global scope items are allocated memory. From the above example:

    `name`: Allocated memory (initialized as `uninitialized` in the ***Temporal Dead Zone**** due to `let`).

    `first`: Allocated memory (initialized as `uninitialized` in the ***Temporal Dead Zone*** due to `const`).

    `second`: The complete function definition is stored directly in memory.

    `x`: Allocated memory and immediately initialized with the value `undefined`.

    > Observe that only memory is allocated before execution.

    &nbsp;


3. Execution of Global code starts:

    `const name = "Rahul";` --> Value `"Rahul"` is assigned to `name`.

    `const first = function() {...};` --> The function is assigned to `first`.

    `var x = first();` --> Since this is a function invocation, the engine pauses the execution of GEC, creates and pushes the Execution Context of `first()` onto the Call Stack and it then starts executing the function `first()`.
   ```
    ||||||||||||||||||||||||||    +++++++++++++++++
    |        GLOBAL EC       |    |  Call-Stack   |
    ||||||||||||||||||||||||||    +++++++++++++++++
    |                        |    |               |
    | name = "Rahul"         |    |               |
    | first = <function>     |    |               |
    | second = <function>    |    |               |
    | x = undefined          |    |               |
    |                        |    |      GEC      |
    ||||||||||||||||||||||||||    +++++++++++++++++
    ```

&nbsp;

4. Execution of Function Code starts. `first()` is pushed to the stack and run.

    Just like the GEC, the values and declaration are processed.
   ```
    ||||||||||||||||||||||||||    +++++++++++++++++
    |        first()         |    |  Call-Stack   |
    ||||||||||||||||||||||||||    +++++++++++++++++
    |                        |    |               |
    | a = 1                  |    |               |
    | b = second()           |    |               |
    | ... paused until       |    |               |
    | second() finishes      |    |   [first()]   |
    |                        |    |      GEC      |
    ||||||||||||||||||||||||||    +++++++++++++++++
    ```

    `second()` is pushed on the call stack for execution and it sets `c = 2`

    ```
    ||||||||||||||||||||||||||    +++++++++++++++++
    |        second()        |    |  Call-Stack   |
    ||||||||||||||||||||||||||    +++++++++++++++++
    |                        |    |               |
    | c = 2                  |    |               |
    |                        |    |               |
    |                        |    |  [second()]   |
    |                        |    |   [first()]   |
    |                        |    |      GEC      |
    ||||||||||||||||||||||||||    +++++++++++++++++
    ```

    `second()` returns `c` and is popped off the call stack and `first()` resumes

    ```
    ||||||||||||||||||||||||||    +++++++++++++++++
    |        first()         |    |  Call-Stack   |
    ||||||||||||||||||||||||||    +++++++++++++++++
    |                        |    |               |
    | a = 1                  |    |               |
    | b = second()           |    |               |
    | a += b                 |    |               |
    |                        |    |   [first()]   |
    |                        |    |      GEC      |
    ||||||||||||||||||||||||||    +++++++++++++++++
    ```
    `first()` resumes, assigns `b` the returned value of `second()` i.e.`2`, updates a to `3` _(a += b)_, returns `3`, and pops off.

    Control returns to the Global Execution Context to finish the last line:

    ```
    ||||||||||||||||||||||||||    +++++++++++++++++
    |        GLOBAL EC       |    |  Call-Stack   |
    ||||||||||||||||||||||||||    +++++++++++++++++
    |                        |    |               |
    | name = "Rahul"         |    |               |
    | first = <function>     |    |               |
    | second = <function>    |    |               |
    | x = first()            |    |               |
    |                        |    |      GEC      |
    ||||||||||||||||||||||||||    +++++++++++++++++
    ```

    `var x = first();` --> The engine takes the returned value `3` and overwrites the initial `undefined` value stored in `x`.

    ```
    ||||||||||||||||||||||||||    +++++++++++++++++
    |        GLOBAL EC       |    |  Call-Stack   |
    ||||||||||||||||||||||||||    +++++++++++++++++
    |                        |    |               |
    | name = "Rahul"         |    |               |
    | first = <function>     |    |               |
    | second = <function>    |    |               |
    | x = 3                  |    |               |
    |                        |    |               |
    ||||||||||||||||||||||||||    +++++++++++++++++
    ```

&nbsp;

For each function call, a NEW Execution Context is created and STACKED ON TOP of the Global Execution Context.

All function execution contexts in sequence together form a call stack.
