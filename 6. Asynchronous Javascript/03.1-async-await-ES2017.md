## `Async/Await`

### `async-await` is simply a syntactic sugar to the `then()` method.

Inside an `async` function, we can have one or more `await` statements

>Syntax:
>
> await `<Promise Object>`

```javascript
const myFunc = async function(param){

  await fetch(".../api/users?page=2");
  // This awaits for the result from the
  // promise untill the data has been
  // fetched.
  // But it doesn't block the main
  // thread
};

```
> Does this block the execution?
>
> No, because this is running asynchronously in the background.

> This also makes our code(function) look clean like a regular function.

Once the result from await is achieved, it can be stored to a variable.

> The above fn can be re-written as:

```javascript
const myFunc = async function(page){
  const response = await fetch(".../api/users?page="+page);

  const data = await response.json();
  // json() also returns a promise

  console.log(val);
};

myFunc(2);

console.log("This will be printed first.");

// Output:
// "This will be printed first."
// Promise {<pending>}
// {...} -  printed once data is fetched
// from the API.
```

`async-await` is simply a syntactic sugar to the `then()` method.

The above example can be reviewed as :
```javascript
const myFunc = async function(page){
  fetch(".../api/users?page="+page)
  .then(val=>val.json()))
  .then(val=>console.log(val));
};
```

> ### IMP: Try-Catch Block using async-await.

> Why try catch?
>
> With `then()` method, we can chain the `catch()`.
>
> However, we cannot chain `catch()` method to an `await` statement.

> Example: Always use `try-catch` block to catch errors in async await.

```javascript
// Psuedo code
try {
  await ...

  if(something_fails)
    throw new Error("Custom Error");

  await ...
  await ...

} catch(err) {
  console.log(err.message);
}
```

This is same as chaining `catch()` to the `then()` as we saw in previous examples.
