## Chained Promises [ES6]

Promise is an object that is used as a placeholder for the future result of an asynchronous operation.

> Example 1:
```javascript
const getData = (page) => {
  fetch(".../api/users?page="+page)
  .then(response => response.json())
  .then((data) => {
    const input2 = data[0];
    if (!input2) {
      return;
    }

    return fetch(".../api/somedata/"+input2);
    // This will again return a promise.
  })
  .then(response => response.json())
  .then(data => console.log(data));
};

getData(2);
```

But But But, callback hell is still possible in this scenario and still it will be difficult to guess, which promise it didnt go through. And thats when we `Handle Rejected Promises`.

&nbsp;

### Handling Rejected Promises.

> #### Noob Way

```
Syntax:
then(
   <callback_for_fulfilled_promise>,
   <callback_for_rejected_promise>
)

e.g.
then(
  data => console.log(data),
  err => console.error(err)
);
```
```javascript
const getDataErrHandled = (page) => {
  fetch(".../api/users?page="+page)
  .then(response => response.json(), err => console.error(err))
  .then(
    (data) => {
      const input2 = data[0];
      if (!input2) {
        return;
      }
      return fetch("https://xyz.com/api/somedata/"+input2);
    }, (err) => console.error(err)
  )
  .then(response => response.json(), (err) => console.error(err))
  .then(data => console.log(data), (err) => console.error(err));
};

getDataErrHandled(2);
```
Now we can find where our code is problematic, if anything goes wrong.

> #### Smart Way - using `catch()`

In above example, writing `(err) => console.error(err)` again and again in every `then()` is a pain and makes the code cluttered.

We can write all the error handling at the end of our _promises chain_ using `catch()` method. It catches error at every stage of the promise chain.



> Example: catch(`<callback to handle rejected promises in a chain.>`)
```javascript
const getDataWithCatch = (page) => {
  fetch(".../api/users?page="+page)
  .then(response => response.json())
  .then((data) => {
    const input2 = data[0];
    if (!input2) {
      return;
    }
    return fetch(".../api/somedata/"+input2);
  })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err.message));
  //Every "Error Object" has a property
  // called 'message'.
};

getDataWithCatch(2);
```

> #### finally()
Always called at the end no matter whether promise is fulfilled or rejected.
```javascript
const getDataWithFinally = (page) => {
  fetch(".../api/users?page="+page)
  .then(response => response.json())
  .then((data) => {
    const input2 = data[0];
    if (!input2) {
      return;
    }
    return fetch(".../api/somedata/"+input2);
  })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err.message))
  //Every "Error Object" has a property
  // called 'message'.
  .finally(()=>{
    // Always executes
    hideLoadingSpinner();
    // Useful to hide the loading gif/img
    // after the ajax/fetch is finished.
   });
};

getDataWithFinally(2);
```

### Reject Promises Manually by throwing errors.

Sometimes the error `404` is also considered as a fulfilled promise. because its a proper response given by server and it is not caught by catch() and the promise is not rejected.

> Example: We can manually throw errors, to reject promises manually in certain cases like 404.
```javascript
const getDataWithFinally = (page) => {
  fetch(".../api/users?page="+page)
  .then(
    response => {
      console.log(response.status); // 404

      if(!response.ok){
        // "ok" property of response of
        // fetch() is either "true" on success
        // or "false" otherwise.

        throw new Error("Failed" + response.status);
        // Error() above is a constructor
        // function to create Error Object
        // in javascript.

        // Now if this error is thrown, this
        // promise will be immediately rejected
        // and the enclosing then()'s promise
        // will be rejected.

        // This directly propogates down
        // to the catch().
      }
      response.json()
  })
  .then((data) => {
    const input2 = data[0];
    if (!input2) {
      return;
    }
    return fetch(".../api/somedata/"+input2);
  })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err.message))
  //Every "Error Object" has a property
  // called 'message'.
  .finally(()=>{
    // Always executes
    hideLoadingSpinner();
    // Useful to hide the loading gif/img
    // after the ajax/fetch is finished.
   });
};

getDataWithFinally(2000);
```
