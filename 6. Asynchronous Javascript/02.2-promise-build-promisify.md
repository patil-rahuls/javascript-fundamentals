## Building Promise

**Promise is an object that is used as a placeholder for the future result of an asynchronous operation.**

### 1. Creating a new Promise

Created using Promise Constructor.
It takes exactly one argument the `executor` function.
This `executor` function has 2 parameters

- `resolve( <to be fulfilled value> )`
- `reject( <rejection reason> )`
  
> Example: Building a Promise
> ```javascript
> const myPromise = new Promise(
>   // executor function
>   (resolve, reject) => {
>     let dataReceived;
> 
>     // ...
>     // some asynchronous operation here
>     // that updates the boolean 'dataReceived'
>     // to either true or false...
> 
>     if (dataReceived) {
>       // Fulfill the promise if 
>       // 'dataReceived' is true.
> 
>       // Code for fulfilled 
>       // promise goes here.
>       resolve("Data fetched successfully!");
>     } else {
>       // Reject the promise if 
>       // 'dataReceived' is false.
> 
>       // Code for Rejected 
>       // Promise goes here.
> 
>       reject("Failed to fetch data.");
>     }
> 
>     // If there's an error, 
>     // call reject('Error Message/Value')
>   },
> );
> ```
> 
> _In the above example:_
> 
> _The `resolve()` function is called to set the Promise as _`fulfilled`_ and the _`<to be fulfilled value>`_ is passed to it:_
> 
> ```javascript
> resolve("Data fetched successfully!");
> ```
> 
> _The passed string `"Data fetched successfully!"` will be available as the _`fulfilled value`_ of that Promise._
>
> _The `reject()` function is called to set the Promise as _`rejected`_ and the Error Message/Value is passed to it:_
> 
> ```javascript
> reject("Failed to fetch data.");
> ```
> _The string `"Failed to fetch data."` will be available as a `rejection reason` of that promise._

------
&nbsp;
### 2. Consuming the Promise - `.then()`

> ```javascript
> myPromise
>   .then((result) => console.log(result))
>   .catch((err) => console.error(err));
> 
> // Output:
> // 'Data fetched successfully!'
> // this gets printed if the 
> // promise is fulfilled.
> ```
> 
> _In the above example, the callback inside the `.then()` is called when the promise resolves._
>
> _And its parameter `result` will hold the `fulfilled value`. The callback inside the `catch()` is called when the promise rejects. And its parameter `err` will hold the `rejection reason`._
>

### 3. Promisifying:

Converting callback based asynchronous behaviour into promise based.

> Example: Promisifying `setTimeout()`
> ```javascript
> const waitForNSecond = function (seconds) {
>   return new Promise(function (resolve) {
>     setTimeout(resolve, seconds * 1000);
>   });
> };
> 
> // Using arrow fn.
> const waitForNSeconds = (seconds) =>
>   new Promise((resolve) => setTimeout(resolve, seconds * 1000));
> 
> // Usage:
> waitForNSeconds(4).then(function () {
>   console.log("I waited for 4 Seconds.");
> });
> 
> // Nested Calls :
> waitForNSeconds(4)
>   .then(function () {
>     console.log("I waited for 4 Seconds.");
> 
>     // This will return another promise.
>     waitForNSeconds(1);
>   })
>   .then(() => {
>     console.log("I waited for another 1 Second");
>   });
> ```
> 
> _In the above example, by chaining the `.then()` methods we prevents the callback hell, also the code looks readable._

Example: Build a Promise and Resolve immediately.
**_Promise.resolve().then()..._**

```javascript
Promise
  .resolve('abc')
  .then((x)=>{ console.log(x) }); 
  // abc

Promise
  .reject(new Error('something failed')).
  .then((x)=>{ console.log(x) }); 
  // Error: something failed

```
