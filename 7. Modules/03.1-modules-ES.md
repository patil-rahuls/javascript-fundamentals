## ES Modules <sup>_ES6_</sup>

They use `import` and `export`.

They are asynchronous, which makes them better for web performance.

Always in Strict Mode

File extension are usually - `.js` (or .mjs)

> **_IMP: In ES Modules (`import`), you get a live connection (also called _live binding_). If the original file updates the value, your imported variable updates automatically._**

### Named Exports/Imports.

A module can have as many named exports as required.

> Exporting Module: `mathUtils.js` (The Provider)
> 
> ```javascript
> const add = (a, b) => a + b;
> const pi = 3.14;
> export { add, pi };
> // OR
> // export individually
> export const add = (a, b) => a + b;
> export const pi = 3.14;
> ```

> Importing Module: `app.js` (The Consumer)
> 
> ```javascript
> import { add, pi } from "./mathUtils.js";
> console.log(add(10, 20));
> // 30
> ```

---

### Default Export/Imports

**A module can only have only ONE default export and more than one named exports.**

_We specify Alias' using `as` keyword._

> Exporting Module: `mathUtils.js` (The Provider)
> 
> ```javascript
> const pi = 3.14;
> const K = -273.15;
> // One default export
> export default (a, b) => a + b;
> // Exporting Alias
> export { pi, K as KelvinConstant };
> ```

> Importing Module: `app.js` (The Consumer)
> 
> ```javascript
> import // Default
> add, {
>   // We can't use 'K' here.
>   KelvinConstant,
>   // Alias while importing.
>   pi as PiConstant,
> } from "./mathUtils.js";
> console.log(add(10, 20));
> // 30
> ```

_When no curly braces are given around, it checks for the default export. e.g. `add` in the case above._

### Namespace Imports

Consider the exporting `mathUtils.js` file from previous example for this example.

Using `*` and `as` we can import everything in an alias.

```javascript
import * as name from "./mathUtils.js";

// `default` references the default exported fn.
const result = name.default(10, 20);

console.log(result);
// 30

console.log(name.pi);
// 3.14
```

_To Summarise -_

- ***Named import:***
  ```javascript
  import {itm1, itm2 } from "module";
  ```

- ***Default import:***
  ```javascript
  import defaultFn from "module";
  ```

- Namespace import:
  ```javascript
  import * as name from "module";
  ```
