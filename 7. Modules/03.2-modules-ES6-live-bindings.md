## ES6 Modules

> #### IMP: Imports are live connection to exports

> In ES6 Modules (`import`), you get a live connection (also called ***live binding***). If the original file updates the value, your imported variable updates automatically.

> IMP: The live-connection is ONE-WAY.
>
> - The Provider (the file that exports) can change the value.
>
> - The Consumer (the file that imports) cannot change it.

&nbsp;

> Exporting Module: `mathUtils.js` (The Provider)
```javascript
export let count = 0;

export function increment() {
  count++;
}
```

> Importing Module: `app.js` (The Consumer)

```javascript
import {
  count,
  increment
} from './counter.js';

console.log(count);
// 0

increment();
console.log(count);
// 1 (It updated!)

increment();
console.log(count);
// 2 (It updated again!)
```

&nbsp;

> IMP Note: The imported values are read-only views of the features that were exported.

> Similar to `const` variables, you cannot re-assign the variable that was imported.

In the above example, in the `app.js` file, if you try to change/re-assign `count`, it will Error out!

```javascript
count = 10
// TypeError: Assignment to constant variable.
```

You can only change it by calling a function (`increment`) that lives inside the original module.

But you can still modify properties of objects.

> The value can only be re-assigned by the module which is exporting it.

Damn! What a feature!
